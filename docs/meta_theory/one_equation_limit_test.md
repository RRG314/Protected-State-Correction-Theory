# One-Equation Limit Test

Date: 2026-04-17

## Question

Does current evidence justify saying “one equation is impossible,” or only a narrower mathematical claim about compressed representations?

## Precise Analogue Tested

Given a system class `C`, can exactness be classified by a single compressed descriptor
`phi(S)`
that omits branch-specific compatibility structure?

Examples of compressed descriptors:
- rank/count/budget only,
- one residual scalar only,
- one operator class label only.

## Evidence Cases

### Case 1: Same-rank opposite verdict
- Restricted-linear branch has opposite exactness at same rank metadata.
- Implication: rank-only `phi` fails.

### Case 2: Same-count opposite verdict
- Soliton lane has same-count observation families with opposite identifiability verdicts (validated on supported family).
- Implication: count-only `phi` likely fails in nonlinear restricted setting.

### Case 3: One operator preserves, another destroys
- Projection/reduction split in soliton and bounded-domain projection mismatch in OCP.
- Implication: operator label alone is too coarse.

### Case 4: Exact on small family, false on enlarged family
- Family-enlargement false-positive theorem.
- Implication: a classifier ignoring admissible-family scope fails.

### Case 5: Exact in one domain, impossible in another
- Periodic exact projection vs bounded-domain incompatibility no-go.
- Implication: domain compatibility cannot be compressed away.

## Does this justify “one equation is impossible”?

No.

Current evidence supports a narrower statement:
- **One compressed representation can be insufficient to preserve target-relevant distinctions needed for exactness.**

It does **not** support:
- universal impossibility of single-equation physical theories,
- broad claims about GR/QM unification impossibility.

## Strongest Defensible Version

`Branch-limited insufficiency claim`:
In heterogeneous recoverability/correction branches, any classifier that ignores compatibility structure (family, symmetry, domain, operator interaction) is generally incomplete and can be provably wrong on supported classes.

Status:
- `PROVED ON SUPPORTED FAMILY` for key anti-classifier and enlargement/mismatch packages.
- `CONDITIONAL` as broader cross-branch meta-statement.

## Falsification target

A true cross-branch scalar/classifier with no structural compatibility inputs that correctly predicts exactness would refute this claim.

## Overview

This release hardens the repository for review and publication flow. The technical expansion from the prior cycle remains intact, but the public surface is now cleaner, the authority map is clearer, and release discipline is explicit.

## What Changed Since v1.1.0

The repository was reorganized around a six-layer architecture:

- theorem core (adopted backbone and canonical spines)
- restricted results (branch-limited contributions)
- methods and diagnostics
- validation and evidence
- physics translation
- meta-governance

Competing authority paths were demoted, internal pass logs were moved out of the front door, and public navigation now points to one canonical route.

## Documentation and Writing Hardening

Active docs were normalized to a research-note style:

- reduced onboarding/product-style prose
- clearer scope labels and claim boundaries
- stronger separation between theorem claims, methods, and interpretation
- removal of machine-specific absolute repo paths in active documentation

## Reviewer-Facing Control Artifacts

Two governance docs were added to support push and release decisions:

- `docs/meta-governance/live-deployed-vs-local-audit-report.md`
- `docs/meta-governance/push-readiness-execution-report.md`

These track branch delta, diff classes, validation gates, exclusions, and remaining release risks.

## Breaking Changes

No API-breaking code changes are introduced in this release.

Repository navigation and document placement changed significantly. Compatibility redirects are retained where needed.

## Validation Snapshot

Validation passed in the repository virtual environment:

- `source .venv/bin/activate && bash scripts/validate/run_all.sh`
- `git diff --check`

Note: system Python may fail without local dependencies (for example `numpy`), so `.venv` is the required execution context for full validation.

## Known Limits

- The PR/review surface remains large because this release bundles structural rehome, writing normalization, and governance hardening over the prior release.
- Internal governance material is intentionally preserved for provenance and may appear dense in tree browsing.

# SDS Scope Boundary (MP-7)

Date: 2026-04-19

## Current boundary decision

SDS language is retained only as:
- engineering layer,
- workflow/diagnostic motif,
- analogy support.

SDS is not promoted as theorem-core unless a claim is explicitly mapped to a theorem node with formal assumptions and proof/evidence status.

## Admission rule for future SDS claims

A new SDS statement may be promoted above `ANALOGY ONLY` only if all are present:
1. formal statement with declared admissible family,
2. explicit map to existing theorem object (`M`, `T`, fibers, or declared operator criterion),
3. proof status label (`PROVED`, `VALIDATED`, `CONDITIONAL`, etc.),
4. falsification witness attempt and outcome.

If any item is missing, status remains `ANALOGY ONLY`.

## Current status

- No SDS theorem-core promotions were added in this pass.
- Existing audit recommendation (demote SDS from core theorem language) is preserved.

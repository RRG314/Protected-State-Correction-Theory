# How to Cite This Work

This repository is meant to be citable both as a research-program repository and as a source of specific theorem, no-go, and application documents.

## Canonical Repository Citation

The canonical machine-readable citation is in:

- [CITATION.cff](../../CITATION.cff)

A BibTeX version is available in:

- [protected-state-correction.bib](./protected-state-correction.bib)

## Recommended Citation Practice

### 1. If you use the repository as a whole

Cite the repository itself.

This is the right choice if you are drawing on the overall framework, the repository architecture, the workbench, or the research program as a package.

### 2. If you use a specific result

Cite both:

- the repository itself, and
- the specific theorem, no-go result, or application document you are relying on

Examples:

- [theorem-spine-final.md](../finalization/theorem-spine-final.md)
- [no-go-spine-final.md](../finalization/no-go-spine-final.md)
- [sector-recovery-theorems.md](../theorem-candidates/sector-recovery-theorems.md)
- [generator-theorems.md](../theorem-candidates/generator-theorems.md)
- [cfd-projection-results.md](../theorem-candidates/cfd-projection-results.md)

### 3. If you use one of the outside anchor directions

Also cite the standard outside source relevant to that branch.

Examples:

- Knill-Laflamme for exact sector recovery and QEC framing
- Chorin or later projection-method literature for exact incompressible/projection branches
- Dedner or related constraint-cleaning papers for asymptotic cleaning branches
- Gundlach and related constraint-damping papers for asymptotic PDE correction

## Suggested Text Citation Style

A clean prose style is:

> Reid, *Protected-State Correction Theory* (repository, version or access date), together with the specific theorem or application document used.

If you are discussing a standard external anchor, add the relevant outside citation in the same sentence or paragraph.

## BibTeX

### Repository citation

```bibtex
@software{reid_protected_state_correction_2026,
  author  = {Reid, Steven},
  title   = {Protected-State Correction Theory},
  year    = {2026},
  version = {1.0.0},
  url     = {https://github.com/RRG314/Protected-State-Correction-Theory},
  note    = {Research program repository. ORCID: https://orcid.org/0009-0003-9132-3410}
}
```

### Notes on versions

If you cite a specific public release, use that tagged release in preference to a floating main-branch citation.

If you cite the current repository state rather than a release tag, include an access date.

# Advanced No-Go Results

## Plain-Language Summary

The strongest way to make the OCP program more finished is not to force more positive claims. It is to sharpen the boundaries.

This file records the stronger failure statements developed after the initial theorem spine.

## No-Go OCP-N7: No Finite-Time Exact Recovery In The Smooth Linear Flow Branch

Let `H` be a finite-dimensional vector space, let `S` and `D` be subspaces with `D ≠ {0}`, and let

```text
Φ_t = e^{-tK}
```

be the flow generated by a linear matrix `K`.

Then for every finite time `t > 0`, the map `Φ_t` cannot satisfy

```text
Φ_t(s + d) = s
```

for all `s ∈ S`, `d ∈ D`.

### Proof

For each finite `t`, the matrix exponential `e^{-tK}` is invertible.

If exact recovery held at time `t`, then setting `s = 0` would give

```text
Φ_t(d) = 0
```

for every `d ∈ D`.

Since `D` is nontrivial, that would place a nonzero vector in the kernel of `Φ_t`, contradicting invertibility.

So a smooth linear flow cannot perform exact one-shot recovery on a nontrivial disturbance family at any finite time.

### Consequence

This is one of the cleanest structural boundaries in the repository:
- exact recovery in the continuous branch needs a projection, reset, measurement, or other non-flow step,
- pure smooth linear damping belongs to the asymptotic branch.

## No-Go OCP-N8: Sector Overlap Destroys Unique Exact Detection

Let `D_i` and `D_j` be two candidate disturbance sectors with

```text
D_i ∩ D_j ≠ {0}.
```

Then there is no pair of linear detectors `Q_i`, `Q_j` satisfying

```text
Q_i x = x for x ∈ D_i,
Q_i x = 0 for x ∈ D_j,
Q_j x = x for x ∈ D_j,
Q_j x = 0 for x ∈ D_i.
```

### Proof

Pick `0 ≠ v ∈ D_i ∩ D_j`.

Then the detector requirements force

```text
Q_i v = v
```

because `v ∈ D_i`, and also

```text
Q_i v = 0
```

because `v ∈ D_j`.

This is impossible. The same contradiction holds for `Q_j`.

So nontrivial sector overlap rules out unique exact sector detection.

## No-Go OCP-N9: Boundary-Sensitive Exact Projection Is Not Yet Established

This is not a theorem, but an explicit repository boundary.

The periodic Leray/Helmholtz projector is exact in the tested branch because the decomposition is clean and the harmonic ambiguity is suppressed by the domain choice.

For boundary-sensitive domains, the repository does **not** yet promote an exact OCP theorem because:
- the decomposition must account for boundary conditions,
- harmonic components may have to be assigned explicitly,
- and the simple periodic proof architecture no longer transfers automatically.

Status:
- `OPEN BOUNDARY`, not a promoted theorem.

## Why These Results Matter

These advanced no-go results improve the program materially because they prevent three common forms of overreach:
- pretending smooth damping can secretly become exact recovery at finite time,
- pretending overlapping sectors can still be uniquely corrected,
- and pretending the periodic projector theorem automatically extends to harder PDE settings.

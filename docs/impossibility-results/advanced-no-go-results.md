# Advanced No-Go Results

## Plain-Language Summary

The strongest way to make the OCP program more finished is not to force more positive claims. It is to sharpen the boundaries.

This file records the stronger failure statements developed after the initial theorem spine.

## No-Go OCP-N7: No Finite-Time Exact Recovery In The Smooth Linear Flow Branch

Let `H` be a finite-dimensional vector space, let `S` and `D` be subspaces with `D ≠ {0}`, and let

```text
Φ_t = e^{-tK}
```

be the flow generated by a linear matrix `K`.

Then for every finite time `t > 0`, the map `Φ_t` cannot satisfy

```text
Φ_t(s + d) = s
```

for all `s ∈ S`, `d ∈ D`.

### Proof

For each finite `t`, the matrix exponential `e^{-tK}` is invertible.

If exact recovery held at time `t`, then setting `s = 0` would give

```text
Φ_t(d) = 0
```

for every `d ∈ D`.

Since `D` is nontrivial, that would place a nonzero vector in the kernel of `Φ_t`, contradicting invertibility.

So a smooth linear flow cannot perform exact one-shot recovery on a nontrivial disturbance family at any finite time.

### Consequence

This is one of the cleanest structural boundaries in the repository:
- exact recovery in the continuous branch needs a projection, reset, measurement, or other non-flow step,
- pure smooth linear damping belongs to the asymptotic branch.

## No-Go OCP-N8: Sector Overlap Destroys Unique Exact Detection

Let `D_i` and `D_j` be two candidate disturbance sectors with

```text
D_i ∩ D_j ≠ {0}.
```

Then there is no pair of linear detectors `Q_i`, `Q_j` satisfying

```text
Q_i x = x for x ∈ D_i,
Q_i x = 0 for x ∈ D_j,
Q_j x = x for x ∈ D_j,
Q_j x = 0 for x ∈ D_i.
```

### Proof

Pick `0 ≠ v ∈ D_i ∩ D_j`.

Then the detector requirements force

```text
Q_i v = v
```

because `v ∈ D_i`, and also

```text
Q_i v = 0
```

because `v ∈ D_j`.

This is impossible. The same contradiction holds for `Q_j`.

So nontrivial sector overlap rules out unique exact sector detection.

## No-Go OCP-N9: Naive Periodic Projector Transplant Fails On A Bounded-Domain Protected Class

This is now sharper than an open caution. It is an explicit rejected bridge.

Take the periodic FFT-based projector used in the exact continuous branch and apply it unchanged to a bounded-domain problem whose protected class includes both:
- divergence-free structure,
- and the intended boundary-normal trace condition.

Then the transplant can fail even when the divergence is driven essentially to zero.

### Repository Counterexample

The repository includes a tested counterexample built from:
- a bounded-domain physical field generated by a stream function, so the boundary-normal trace is essentially zero,
- a nonperiodic gradient contamination,
- and the periodic projector applied anyway.

Observed outcome in the tracked example:
- post-projection divergence drops to machine-scale,
- but the projected field acquires a nonzero boundary-normal trace,
- so it is not an exact correction operator for the bounded-domain protected class.

### Consequence

This does **not** prove that bounded-domain exact correction is impossible.

It proves something narrower and still important:

> the periodic exact projector cannot simply be transplanted unchanged and still count as an exact bounded-domain correction operator.

Status:
- `DISPROVED` for the naive transplant claim,
- bounded-domain exact correction itself remains `OPEN`.

## No-Go OCP-N10: Divergence-Only Recovery Cannot Exactly Recover A Nontrivial Bounded Protected Class

Let `S_bc` be a bounded incompressible protected class containing two distinct states `u_1`, `u_2` with the same divergence data, and suppose a candidate correction architecture factors only through the divergence scalar field:

```text
R(u) = F(div u).
```

Then `R` cannot be an exact recovery operator on `S_bc`.

### Proof

Because `u_1` and `u_2` have the same divergence data,

```text
div u_1 = div u_2.
```

So the divergence-only architecture gives

```text
R(u_1) = F(div u_1) = F(div u_2) = R(u_2).
```

But exact recovery would require

```text
R(u_1) = u_1,
R(u_2) = u_2.
```

Since `u_1 ≠ u_2`, this is impossible.

### Consequence

This result sharpens the CFD boundary:
- removing divergence alone is not enough when the protected class contains richer velocity information,
- bounded-domain exact correction must encode more than a scalar constraint residual,
- and exact pressure projection on bounded domains must be understood through the full domain-compatible Hodge projector, not through a divergence-only heuristic.

## Why These Results Matter

These advanced no-go results improve the program materially because they prevent three common forms of overreach:
- pretending smooth damping can secretly become exact recovery at finite time,
- pretending overlapping sectors can still be uniquely corrected,
- and pretending the periodic projector theorem automatically extends unchanged to harder PDE settings.

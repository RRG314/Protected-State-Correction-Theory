# Structural Discovery Algorithms

The Structural Discovery subsystem has four main algorithmic layers.

## 1. Structural Failure Detector

Restricted-linear detector:

- compute recoverable and unrecoverable protected rows
- compute nullspace witness and protected gap
- compute minimal candidate-library augmentation sets
- compute unrestricted rank-deficiency augmentation count

Periodic/modal detector:

- evaluate current record family (`full_vorticity`, `cutoff_vorticity`, `divergence_only`)
- compute current threshold status for the chosen protected target
- identify weaker protected targets already visible under the retained support

Diagonal/history detector:

- compute minimal exact horizon for the chosen protected functional
- distinguish finite-history insufficiency from hidden-functional impossibility
- detect weaker functionals already generated by the current sensed moment family

Qubit detector:

- compute fixed-basis collision law for the chosen phase window
- separate stronger and weaker protected targets

## 2. Minimal Augmentation Engine

The augmentation layer tries to produce the smallest meaningful fix.

Current supported exact or validated cases:

- restricted-linear candidate row addition
- periodic cutoff increase
- diagonal history increase
- weaker-target switch in the qubit split

When the repo does not yet support an exact fix theorem, the subsystem either:

- returns a documented heuristic suggestion, or
- explicitly says no validated in-studio fix exists.

## 3. Structural Redesign Engine

Recommendations are ranked by three questions:

1. Is the proposed change theorem-backed or family-backed?
2. Is it minimal inside the current branch?
3. Does it move the current setup into an exact, asymptotic, or still-failing regime?

Current action kinds:

- `add_measurement`
- `add_mode`
- `add_history`
- `weaken_target`
- `switch_record`
- `switch_architecture`
- `keep`

## 4. Comparison Engine

The comparison layer runs a before/after check on a chosen recommendation.

For each validated recommendation the system records:

- before regime
- after regime
- whether the regime changed
- key metric before and after
- short narrative explaining why the change worked

## Demo Bundle

The reproducible demo bundle is built by `scripts/compare/run_structural_discovery_examples.py` and currently includes:

- periodic modal repair
- control history repair
- weaker-vs-stronger target split
- restricted-linear measurement repair

## Design Influences

The subsystem is intentionally closer to engineering diagnosis and sensor-placement tooling than to generic theorem browsers.

Relevant outside anchors:

- functional observability and sensor selection work in control
- model-based fault diagnosis and isolability-oriented sensor placement
- decision tools that show failure reason, repair candidate, and evidence side by side

The repo uses those patterns as interface guidance, not as novelty claims.

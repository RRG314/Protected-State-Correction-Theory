# Structural Discovery Demo Walkthroughs

## Demo 1 — Periodic Modal Repair

Start:

- protected target: `full weighted modal sum`
- record: `cutoff vorticity`
- cutoff: `3`

Diagnosis:

- the retained cutoff misses part of the protected support
- weaker lower-support targets remain recoverable

Suggested fix:

- raise the cutoff to `4`

Outcome:

- regime changes from `impossible` to `exact`
- collision gap drops to zero on the tested family

## Demo 2 — Control History Repair

Start:

- family: diagonal three-state scalar-output model
- protected target: `second sensor moment functional`
- horizon: `2`

Diagnosis:

- the finite history is too short to interpolate the target on the active sensor spectrum

Suggested fix:

- increase horizon to `3`

Outcome:

- regime changes from `impossible` to `exact`

## Demo 3 — Weaker-Vs-Stronger Target Split

Start:

- family: fixed-basis qubit record
- protected target: full Bloch vector
- phase window: `30°`

Diagnosis:

- phase freedom creates fiber collisions under the fixed-basis record

Suggested fix:

- weaken target to `z coordinate only`

Outcome:

- regime changes from `impossible` to `exact`
- the studio also shows a richer-basis recommendation as a standard external path rather than as a repo theorem

## Demo 4 — Boundary Architecture Repair

Start:

- family: bounded-domain projection architecture test
- current architecture: periodic projector transplant
- protected target: full bounded protected class

Diagnosis:

- divergence is reduced, but the boundary-normal trace is wrong
- the issue is architectural, not just numerical

Suggested fix:

- switch to the restricted boundary-compatible finite-mode Hodge projector

Outcome:

- regime changes from `impossible` to `exact`
- the demo makes the repo’s bounded-domain scope control visible inside the studio instead of only in the docs

## Demo 5 — Restricted-Linear Measurement Repair

Start:

- record rows: `x1` and `x2+x3`
- protected target: `x3`

Diagnosis:

- a nullspace witness changes the protected variable while leaving the current record fixed

Suggested fix:

- add one minimal measurement row from the candidate library

Outcome:

- the current demo selects `measure x2`
- regime changes from `impossible` to `exact`

## Reproducibility

PYTHONPATH=src python3 scripts/compare/run_structural_discovery_examples.py

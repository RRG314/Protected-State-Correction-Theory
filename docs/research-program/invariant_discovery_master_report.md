# Invariant Discovery Master Report

Status: conditional invariant program with tool layer.

This pass tested whether a new universal invariant could displace the existing recoverability backbone. It did not. The stable exact core remains fiber/factorization plus row-space/kernel compatibility.

What survived as additive structure is narrower and useful. Context and augmentation quantities (`CID`, `delta_free`, `delta_C`) and descriptor-lift diagnostics (`DFMI`, `IDELB`, `CL`) provide measurable classification gains on supported families, especially in multi-context and constrained-library settings where amount-only descriptors fail.

The theorem-grade part of this lane is branch-limited. CID-style coherence tests, positive shared augmentation thresholds, and constrained-library infeasibility certificates survive on declared classes. Several other candidates, including broad alignment-like metrics and stress-derived fragility rates, are retained as diagnostics rather than promoted invariants.

The recommended program is a four-layer invariant stack: exact core invariants first, context coherence second, repair deficiency invariants third, descriptor-lift diagnostics fourth. This keeps the lane mathematically useful without overclaiming novelty.

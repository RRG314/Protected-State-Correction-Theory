# SFPR Theorem Candidates

Status: `EXPLORATION / NON-PROMOTED`

This file records theorem-candidate statements tested in the SFPR pass. None are promoted to canonical spine in this pass.

## Scope

All theorem statuses below are for supported finite linear/contextual families generated by `scripts/sfpr/run_sfpr_pass.py` and cross-checked against existing OCP/recoverability formulations.

## A) Structure Collapse Criterion

Statement:
If `M(x)=M(x')` for some pair with `tau(x) != tau(x')`, then exact recovery of `tau` from `M` is impossible on that family.

Status: `PROVED` (supported family) and `LITERATURE-KNOWN`.

Notes:
- This is core collision/fiber logic; additive value is only placement inside SFPR architecture.

## B) Conditioned-vs-Invariant Split

Statement:
There exist context families `{M_c}` such that each context is exactly recoverable for `tau`, but no shared decoder `R_*` is exact across contexts.

Status: `PROVED` (supported family), `PLAUSIBLY DISTINCT` packaging.

Evidence:
- `conditioned_vs_invariant_split_support = 224`.
- Persistence anomalies include `local_exact_global_fail` and `context_drift_flip`.

## C) Positive Augmentation Threshold (Shared Recovery)

Statement:
For some context families, exact shared recovery appears only after a finite augmentation threshold, despite contextwise exactness before augmentation.

Status: `CONDITIONAL`.

Evidence:
- Threshold flips observed (`390`) but augmentation law is empirical in this pass.
- No full symbolic necessity/sufficiency proof yet.

## D) Amount-Only Insufficiency

Statement:
Rank/count/budget descriptors do not classify exact target recoverability.

Status: `PROVED` (supported family), `CLOSE PRIOR ART / REPACKAGED`.

Evidence:
- `same_descriptor_opposite_verdict = 281`.
- Rank predictor mismatches `1347/4560`; budget predictor mismatches `2980/4560`.

## E) Formation-to-Recoverability Bridge

Statement:
There exist formation mechanisms that increase structural organization while not improving exact recoverability for the target.

Status: `VALIDATED / EMPIRICAL ONLY`, `OPEN / NOT READY` for theorem form.

Evidence:
- `formation_without_recoverability = 143`.

## F) Persistence-to-Recoverability Bridge

Statement:
Context/perturbation can preserve local recoverability while destroying context-invariant recoverability.

Status: `PROVED` (supported family), `PLAUSIBLY DISTINCT`.

Evidence:
- `local_exact_global_fail = 224`.
- `context_drift_flip = 223`.

## G) Design Failure Theorem Candidate

Statement:
There exist design instances where global-information criteria (`D`-style) fail exact target recovery while target-alignment criteria succeed.

Status: `VALIDATED / EMPIRICAL ONLY`, `CLOSE PRIOR ART / REPACKAGED`.

Evidence:
- `design_failure_support = 38` conflicts out of `320` design pairs.

## H) Distributed Allocation Theorem Candidate

Statement:
For fixed total budget, allocation geometry can change exact recoverability verdict.

Status: `CONDITIONAL` (strong evidence, proof incomplete), `PLAUSIBLY DISTINCT` on this corpus.

Evidence:
- multi-terminal anomalies and threshold flips; positive/negative `PSI` regimes.

## Candidate Priority for Formal Proof

1. Conditioned-vs-invariant split (`B`) as strict theorem package.
2. Persistence-to-recoverability bridge (`F`) as shared-decoder fragility theorem.
3. Distributed allocation split (`H`) with precise geometry assumptions.

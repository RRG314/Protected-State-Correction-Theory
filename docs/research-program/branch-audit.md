# Branch Audit

This document explains how the repository’s branches fit together, where they are strong, and where they are deliberately limited. It is not a changelog. It is a program-level reading guide.

## How to Read the Program

The program has one foundation and several strengthening lanes.

- The foundation is OCP exact/asymptotic/no-go structure.
- Strengthening lanes test recoverability under constrained observation, anti-classifier limits, family-enlargement risk, and bounded-domain compatibility.
- Tooling lanes expose those results through the workbench with explicit evidence labels.

A branch is kept public-facing when it adds either theorem leverage, reliable falsification leverage, or validated decision value.

## Current High-Value Branches

### Foundational OCP branch
This branch remains the formal anchor. It carries the exact projector/sector structures, asymptotic generator structure, and canonical no-go statements used to keep later branches honest.

### Restricted-linear recoverability branch
This branch is the main route from formal theory to practical diagnosis. It explains when coarse records are sufficient, when they fail, and how minimal augmentation repairs can be computed on supported families.

### Fiber-based recoverability / impossibility branch
This branch gives the cleanest statement of what survives as a universal core (factorization/fiber constancy) and what fails under amount-only reasoning. It is the repository’s primary anti-overclaim layer.

### Descriptor-fiber anti-classifier branch
The Descriptor-fiber anti-classifier branch adds finite-class quantitative diagnostics (`DFMI`, `IDELB`, `CL`) above anti-classifier witnesses. It is useful and active, but intentionally kept branch-limited.

### Adjacent TSIT quantitative extension lane
The TSIT lane is retained as an adjacent quantitative extension, not as a theorem-spine branch. It adds target-specific design/allocation/context diagnostics with strong witness evidence but high overlap with existing OCP row-space/fiber logic, so its placement remains `EXPLORATION / NON-PROMOTED` (Option D).

### Quantum measurement-alignment design lane
This lane is retained as a narrow quantum measurement-design sub-branch. It contributes corrected restricted-class tradeoff identities, explicit failure boundaries, and target-sensitive design diagnostics. It is kept branch-limited because literature overlap is high and no universal novelty claim is supported.

### Bounded-domain / CFD obstruction lane
This lane keeps periodic exactness and bounded-domain failures in the same analytical frame. It is especially important for preventing false transfer from periodic projector results to bounded settings.

### Workbench and composition lanes
Structural Discovery and Discovery Mixer are retained because they expose theorem-backed and validated branch behavior in a reproducible decision workflow. Their value depends on staying aligned with the status map.

## Branches Kept as Conditional or Companion

Soliton overlap is maintained as a bounded conditional bridge. It contributes structural comparisons where explicit overlap survives but is not promoted as repo-wide theorem core. Companion repositories remain the home of record for direct soliton or domain-first development.

## Branches Demoted from Front-Door Priority

Exploratory narrative bundles and repeated process summaries are retained for provenance but demoted from canonical onboarding paths. The criterion is simple: if a document does not improve understanding, validation, or extension, it should not compete with canonical entry docs.

## Audit Conclusion

The repository is strongest when read as one coherent program: foundational OCP structure, branch-limited recoverability strengthening, explicit impossibility boundaries, and theorem-linked tooling. Branch boundaries are not a weakness; they are the mechanism that preserves mathematical honesty.

# Language Cleanup Report

Date: 2026-04-17

Goal: remove weak or inflationary language in the unifying-theory layer and replace it with scope-precise statements.

## 1) Cleanup Rules Applied

1. Replace broad claims with class-scoped statements.
2. Replace suggestive words (`deep`, `complete`, `foundational`, `everything`) with theorem-status language.
3. Replace unlabeled statements with explicit status labels.
4. Force distinction among:
   - universal core,
   - branch-limited theorem package,
   - branch-limited validated package,
   - engineering consequence,
   - analogy-only,
   - rejected.

## 2) Phrase-Class Replacements

| Weak phrase class | Replacement pattern |
|---|---|
| "universal law" | "universal core on declared `(A,T,M)` recoverability formalism" |
| "general theory" | "branch-limited framework with explicit promotion boundaries" |
| "works across all branches" | "holds on declared supported class / remains conditional outside" |
| "structure determines everything" | "structure determines exactness/failure in declared class; amount-only summaries fail there" |
| "projection should preserve" | "operator/domain/family-specific preservation with explicit no-go counterclasses" |
| "emergence is equivalent to correction" | "conceptual parallel only; no theorem transfer" |

## 4) Status-Discipline Cleanup

- Removed use of `DISPROVED` in unifying theorem formation docs.
- Normalized to allowed set:
  - `PROVED`
  - `PROVED ON SUPPORTED FAMILY`
  - `CONDITIONAL`
  - `VALIDATED`
  - `OPEN`
  - `ANALOGY ONLY`
  - `REJECTED`

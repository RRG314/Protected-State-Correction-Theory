# One-Page Summary: Structure-Dependent Recoverability and Failure Theory

Date: 2026-04-17

## What the theory is
A branch-limited framework for deciding when a target can be exactly recovered/preserved, when failure is unavoidable, and when apparent success is fragile.

## What is universal
### Universal Core (promoted)
1. **Exact recoverability = factorization/fiber constancy** on declared admissible family.
2. **Exact failure = target-distinguishing collision** (indistinguishability no-go).

These are theorem-level (`PROVED`) and form the core across the ecosystem.

## What is branch-limited
### Branch-Limited Theorem Package
- no rank-only exact classifier (`PROVED` in supported restricted-linear class),
- no fixed-budget-only exact classifier (`PROVED` in supported restricted-linear class),
- family-enlargement false positives (`PROVED` in supported class),
- model-mismatch instability (`PROVED` in canonical class),
- boundary/domain/topology compatibility obstruction (`PROVED` in declared classes),
- symmetry-quotient obstruction and same-count opposite-verdict restricted analogue (soliton lane: `PROVED` / `PROVED ON SUPPORTED FAMILY` + `CONDITIONAL`).

## Strongest results
- universal recoverability/no-go core,
- anti-classifier theorem package,
- fragility package (family enlargement + mismatch),
- bounded-domain obstruction + restricted exact bounded family,
- restricted nonlinear symmetry/observation no-go analogues.

## What the theory does not claim
- no universal amount-only law,
- no universal projection-preservation law,
- no universal emergence/correction equivalence,
- no theorem-core promotion of SDS/two-reservoir motif,
- no full global MHD closure unification.

## Why it matters
It gives one disciplined way to connect exactness, impossibility, fragility, and repair design across related branches without fake unification. It improves rigor by forcing explicit admissible-family assumptions and status labels on every promoted claim.

## Current status
- Universal core: `PROVED`.
- Strongest branch package: `PROVED` on declared classes.
- Nonlinear and engineering bridges: retained only with explicit restricted or validated status.

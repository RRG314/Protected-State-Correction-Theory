# Reviewer Response Layer

Date: 2026-04-17
Purpose: publication-facing response set to expected criticism.

## R1. "What is actually new here?"

Response:
The universal core is standard in spirit and is not claimed as novel. The distinct contribution is the rigorously status-labeled synthesis: one core theorem/no-go pair, one branch-limited anti-classifier/fragility package, one explicit domain-obstruction lane, and explicit nonclaims preventing false universal promotion.

## R2. "Is this just restricted linear algebra?"

Response:
Restricted-linear theorems are a central proved layer, but the framework contribution is not “row-space theorem exists.” It is the integrated package tying exactness, anti-classifier failure, family enlargement, mismatch instability, and branch-governance into one theorem hierarchy with explicit boundaries.

## R3. "Why call this unified if most results are branch-limited?"

Response:
Because the framework is explicitly branch-limited. Unification is at the object-language and core-theorem level; strengthening is intentionally branch-scoped. This is an explicit design choice, not a hidden weakness.

## R4. "Are validated simulations being passed off as theorems?"

Response:
No. Results are stratified into `U/B1/B2/E/A/R` levels. Only `U` and `B1` are theorem-core. `B2` and `E` are retained for reproducible evidence and engineering consequence without theorem promotion.

## R5. "Projection claims look inconsistent"

Response:
A universal projection law is explicitly rejected. The framework predicts projection outcomes are operator/domain/family-dependent; this resolves the apparent inconsistency.

## R6. "MHD and soliton links look forced"

Response:
Equation-level unification is not claimed. Only structural parallels that survive map-level translation and status tests are retained. Unsupported transfers are marked `ANALOGY ONLY` or `REJECTED`.

## R7. "Where are the hard limits?"

Response:
Limits are explicit in `nonclaims_and_scope_limits.md` and embedded in the framework paper: no universal amount-only law, no universal projection law, no universal emergence-correction equivalence, no theorem-core SDS motif.

## R8. "What would falsify this framework?"

Response:
Any of the following:
1. a valid universal amount-only classifier on declared supported classes,
2. a universal projection-preservation theorem surviving existing counterexamples,
3. a theorem-level transfer of emergence equivalence that closes current branch gaps.

Absent these, branch-limited framing remains the strongest honest position.

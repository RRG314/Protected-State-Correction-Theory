# Exact vs Asymptotic Correction

## Plain-Language Summary

The most important structural distinction in this repository is not between quantum and classical systems. It is between **exact correction** and **asymptotic correction**.

That split makes the theory usable:
- QEC and Helmholtz projection belong on the exact side.
- GLM cleaning and many feedback architectures belong on the asymptotic side.
- A lot of confusion disappears once those are not forced into the same theorem statement.

## 1. Exact Correction

Exact correction means there is a recovery law `R` such that

```text
R(s + d) = s
```

for every admissible protected component `s` and admissible disturbance `d`.

Typical exact cases:
- orthogonal projection `P_S` on `H = S ⊕ D`,
- syndrome-conditioned QEC recovery under Knill-Laflamme assumptions,
- Helmholtz/Leray projection for divergence-free recovery from additive gradient contamination on suitable domains.

Exact correction is strongest when it is:
- one-step,
- unambiguous,
- and leaves `S` fixed exactly.

## 2. Asymptotic Correction

Asymptotic correction means the correction architecture produces a flow or iteration such that

```text
Φ_t(s + d) -> s
```

as `t -> ∞`, while keeping the protected component unchanged or approximately unchanged along the way.

Typical asymptotic cases:
- `x_dot = -k P_D x`,
- GLM divergence cleaning,
- disturbance-damping control laws on invariant error coordinates.

Asymptotic correction matters when exact projection is too expensive, unavailable, or physically unrealistic.

## 3. Why The Split Matters

If exact and asymptotic correction are mixed carelessly, the theory becomes vague. Exact recovery has stronger algebraic requirements:
- clean decomposition,
- distinguishability,
- and an operator that annihilates the disturbance component exactly.

Asymptotic correction can survive with weaker structure:
- a residual signal instead of an exact decomposition,
- contractive evolution instead of one-shot recovery,
- and mode suppression instead of exact annihilation.

## 4. The Bridge

The bridge between the two branches is operator-theoretic.

If `K` satisfies
- `ker(K)=S`,
- `D` invariant under `K`,
- `exp(-tK)|_D` contractive,

then the flow generated by `-K` is an asymptotic correction law.

If additionally `K` is or induces a complementary projector, one can often recover the exact branch in the limit or in a split-step formulation.

The simplest example is:

```text
K = k P_D
```

which gives

```text
x(t) = P_S x(0) + e^{-kt} P_D x(0).
```

## 5. What Does Not Cross Over Cleanly

Not every exact system has a useful asymptotic analog.
Not every asymptotic damping system has an exact projector hiding behind it.

Examples:
- QEC recovery is exact but often measurement-conditioned and sector-indexed, not a single global projector on all of physical Hilbert space.
- GLM cleaning damps constraint error but is not an exact Helmholtz projector.
- Generic controller feedback may suppress deviation while still mixing protected and disturbance coordinates.

## 6. Practical Conclusion

The best current cross-domain statement is:

> OCP is a framework with an exact branch and an asymptotic branch, linked by operator conditions, but not collapsed into one universal correction theorem.

That is narrower than a grand unification claim, but strong enough to support theorem work, no-go work, and practical design rules.

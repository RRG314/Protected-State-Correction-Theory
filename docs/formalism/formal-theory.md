# Formal Theory

## Plain-Language Summary

The Orthogonal Correction Principle becomes mathematically useful only when we stop treating it as a slogan and instead describe an actual correction architecture:
- what must be preserved,
- what counts as disturbance,
- what operator performs correction,
- and what exact or asymptotic property that operator satisfies.

The cleanest exact branch is an orthogonal splitting model. The cleanest asymptotic branch is a contractive flow model. QEC and projection-based divergence cleaning fit the exact side well. GLM cleaning and some control architectures fit the asymptotic side more naturally.

## 1. State-Space Setup

Let `H` be a finite-dimensional inner-product space or a Hilbert space.

We distinguish:
- a **protected subspace** `S \subset H`,
- a **disturbance space** `D \subset H`,
- a **correction structure** `C`, which may be a projector, a recovery map, a feedback generator, or a family of syndrome-conditioned maps,
- and a **recovery law** `R`, discrete or continuous, describing what the correction architecture actually does.

The basic question is:

> Given a disturbed state, can the correction architecture recover or preserve the `S` component without corrupting it?

## 2. Exact OCP System

An **exact OCP system** is a tuple

```text
(H, S, D, R)
```

with the following intended meaning:
- every state under consideration can be written as `x = s + d` with `s \in S` and `d \in D`,
- `R : H -> H` is a recovery map,
- `R(s + d) = s` for all admissible `s,d`.

The cleanest exact case is:

```text
H = S ⊕ D,   S ⟂ D,   R = P_S.
```

Here `P_S` is the orthogonal projector onto `S`.

### Exact Recovery

We say that `R` gives **exact correction** on `(S,D)` if

```text
R|_S = I_S,   R|_D = 0,
```

and therefore `R(s+d)=s` on `S ⊕ D`.

This formulation is deliberately narrow. It excludes cases where correction is only approximate, only probabilistic, or only asymptotic.

## 3. Asymptotic OCP System

An **asymptotic OCP system** is a tuple

```text
(H, S, D, Φ_t)
```

where `Φ_t` is a semigroup or flow and

```text
Φ_t(s + d) = s + η_t(d),   with   η_t(d) -> 0  as  t -> ∞.
```

The protected component is preserved, while the disturbance component is suppressed rather than removed in one step.

The model example is generated by

```text
x_dot = -Kx
```

with
- `ker(K) = S`,
- `D` invariant under `K`,
- and `exp(-tK)|_D` contractive.

The simplest special case is `K = k P_D` with `k > 0`.

## 4. Orthogonality Condition

The phrase “orthogonal correction” should not be used casually. In this repository it means one of the following precise things:

1. **Exact orthogonal splitting:** `S ⟂ D` in the chosen inner product, and correction is projection onto `S`.
2. **Sector orthogonality:** different disturbance sectors are orthogonal to one another, as in syndrome-separated QEC families.
3. **Projection orthogonality:** the unwanted component lies in the image of a complementary projector or gradient field class, as in Helmholtz decomposition.

If none of these hold, the term “orthogonal” is not promoted.

## 5. Distinguishability Condition

Recovery requires more than “error exists.” It requires that the correction architecture can identify the disturbance well enough to act without ambiguity.

In the direct-sum setting, distinguishability is built into the decomposition `H = S ⊕ D`.

In QEC, distinguishability is encoded by orthogonal syndrome sectors or equivalent Knill-Laflamme conditions.

In continuous correction settings, distinguishability is often weaker: one only has access to a projected unphysical mode, a residual, or a damped constraint variable.

## 6. Recoverability Condition

A disturbance family is **recoverable** if there exists a correction law that:
- fixes the protected component,
- removes or suppresses the disturbance family,
- and does so on the admissible state class without ambiguity.

Exact recoverability is stronger than asymptotic correctability.

## 7. Correction Capacity

This program does **not** claim a single universal scalar correction-capacity number across QEC, PDE cleaning, and control.

Instead, the current defensible notion is category-specific:
- finite-dimensional exact branch: `dim(D)` or rank of the disturbance family,
- QEC branch: size/structure of correctable error sectors,
- PDE branch: which constraint-violating modes lie in the image of the correction architecture,
- control branch: which disturbance coordinates are observable and contractible without damaging `S`.

This is enough for design rules, but not yet a universal invariant.

## 8. Failure of Correction

Correction fails nontrivially when at least one of the following happens:
- `S` and `D` are not cleanly separable,
- disturbances are not distinguishable,
- the correction operator acts nontrivially on `S`,
- the disturbance family is larger than the image reachable by the correction architecture,
- or the representation change destroys the geometric meaning of the split.

These are not side comments. They are part of the theory.

## 9. Representation Invariance

OCP statements are invariant under isometries that preserve the geometric data of the problem.

More precisely, if `U : H -> H'` is unitary (or orthogonal in the real case), then

```text
(H, S, D, R)  ->  (H', U(S), U(D), U R U^{-1})
```

preserves exact OCP structure.

This is weaker than saying the framework is basis-free under arbitrary coordinate changes. Orthogonality and projection depend on the inner product or on the transported geometry. That limitation matters.

## 10. Primary Formal Choice

The repository uses the following hierarchy:

1. **Primary formalism:** inner-product / projection-operator language.
2. **Secondary formalism:** syndrome-sector formulation for QEC.
3. **Secondary formalism:** contractive semigroup or feedback-generator language for asymptotic correction.

This choice is deliberate. It keeps the exact theorem spine short and testable while still allowing meaningful continuous-time extensions.

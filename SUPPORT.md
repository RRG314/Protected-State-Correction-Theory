# Support

## Best Way To Get Help

Use GitHub issues for:

- theorem clarification requests
- documentation problems
- workbench bugs
- validation or reproducibility issues
- carefully scoped proposal discussions

## Good Support Requests Include

- the file or module you were using
- what you expected to happen
- what actually happened
- steps to reproduce the issue
- whether the issue is about proof status, documentation, code, or the workbench

## What This Repository Supports Best

This repository is best suited for:

- questions about current theorem scope
- questions about exact versus asymptotic classification
- questions about kept versus rejected bridges
- contributions to validation, docs, and workbench clarity

## What To Avoid

Please avoid posting broad speculative proposals without anchoring them to an existing theorem, operator, no-go result, or application lane.

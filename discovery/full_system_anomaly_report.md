# Full System Anomaly Report

Primary artifact: `data/generated/discovery/full_system_anomaly_catalog.csv` (21 ranked anomalies).

## Ranking Method Used

Each anomaly was ranked by:
1. mathematical clarity,
2. reproducibility,
3. cross-branch significance,
4. direct relevance to exactness/no-go classification.

Scores are in `rank_score` (0–10) and rank order is `rank_position`.

## Repeated Cross-Branch Pattern

The strongest recurrent structure across OCP, context-sensitive, CFD, TSIT, and soliton branches is:

- **descriptor parity with verdict split**

Concretely:
- same rank -> opposite exactness,
- same budget -> opposite exactness,
- same observation dimension -> opposite identifiability,
- same per-context exactness -> opposite shared exactness.

This is the most stable “clear new-results” backbone in the full corpus.

## Threshold/Fragility Pattern

A second recurrent pattern appears in context-sensitive and MHD lanes:

- **small structural changes produce discontinuous exactness behavior**

Examples:
- adding one shared augmentation row restores invariant exactness,
- adding one context flips shared exactness (`1 -> 0`),
- decreasing sheet width in MHD sharply increases localized closure defect.

The anomaly mining confirms one dominant structural message across the system: **amount descriptors are insufficient; compatibility geometry controls exactness and failure**. This is where the strongest publishable and developable material currently sits.
